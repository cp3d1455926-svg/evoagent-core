### 飞书渠道修复

**根因：** FeishuChannel 代码写了但无人调用 + SDK API 过时
- `src/channels/feishu.ts` 定义了完整的 FeishuChannel 类
- 但 CLI 入口和 gateway 从未 import 和启动它
- `--channel feishu` 参数挂在 CLI 选项里但无实际效果

**修复：**
1. CLI index.ts 添加 `startFeishuChannel()` — 读取 env 或 config.yaml 的凭证，WebSocket 连接
2. SDK API 兼容修复 — `@larksuiteoapi/node-sdk` v1.65.0 用 `EventDispatcher.register()` 代替 `.on()` 方法
3. 与 `evoagent gateway` 集成 — gateway 命令自动同时启动网关 + 飞书

### QQ 渠道新增

**新增 `src/channels/qq.ts`**
- 基于 QQ 开放平台 WebSocket API（bot.q.qq.com）
- 支持 C2C_MESSAGE_CREATE 私聊消息
- 自动重连（心跳 + RESUME）
- 配置：QQ_APP_ID + QQ_BOT_TOKEN（env 或 config.yaml）

**与 gateway 集成**
- `evoagent gateway` 自动启动 网关 + 飞书 + QQ
- 没配凭证的渠道自动跳过

### 配置向导更新

- 新增第 5 步：飞书渠道配置（App ID / App Secret / Domain）
- 配置写入 `~/.evoagent/config.yaml`
- CLI 启动时自动读取配置文件
